Version Control

A version control system (VCS) allows you to track the history of a collection of files and supports creating versions of this collection. Each version captures a snapshot of the files at a certain point and the VCS allows you to switch between these versions. Additonally, version control makes edits to files accessible via branching and merging any edits. Also, if you should accidently delete a file, version control has an automatic backup feature. Another benefit of version control is the ability for multiple people to collaboratively edit text files.

How does git help you keep track of changes?

So far in my learning process there are a few key commands that have helped me understand how git helps me track files. To begin, the following commands should be embedded in your memory:

1. Make a new file: touch newfile.txt

2. Add it: git add newfile.txt

3. Make an initial commit: git commit -m "Initial commit"

4. See what you have done: git log

5. Check to see what has changed: git diff

Its a good idea to commit early, and commit often. Commit is the equivalent of "save". If your commits are small, it will become much easier to go back and review what you have done, and allow you to undo the damage you may have caused. So to answer the question, how does git help you track changes, is by letting you commit early and often, and by letting you track all of the changes you made to you program.
